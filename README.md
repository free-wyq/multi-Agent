# Multi-Agent Framework

基于多智能体协作的软件工程虚拟交付平台。

## 项目简介

一个多智能体协作框架，用户可在 Web 页面上创建智能体、配置角色和技能，将智能体拉入群组协作完成软件交付任务。群主智能体负责意图分析和任务调度，子智能体在 Docker 沙箱中执行开发、编译、测试等具体工作。

**核心定位**：先验证可行性，后续再决定是做内部生产工具还是 Demo 平台。

## 核心设计决策

### 1. 两类智能体

| | 群主 | 子智能体 |
|--|------|---------|
| 本质 | 轻量 LLM API 调用 | Claude Code CLI/SDK 实例 |
| 需要沙箱 | ❌ | ✅ Docker 容器 |
| 职责 | 意图分析、任务拆解、DAG调度 | 开发、编译、测试 |
| 运行位置 | 后端进程内 | Docker 容器内 |
| 成本 | 低 | 高 |

### 2. 环境卷分离

容器用完即弃，环境卷保留。下一个智能体挂同一个卷，无需重建环境。

### 3. 技能+镜像+中间件，用户不感知

一个 Ubuntu 基础镜像 + Claude Code CLI，所有角色通用。角色模板自动映射技能，中间件运行时自装自起。用户只选角色，剩下全自动。

### 4. DAG 依赖感知调度

无依赖的任务并行派发，有依赖的等前置完成后再派发。消息队列本身不限制串行或并行。

### 5. 容器内中间件

中间件直接跑在智能体容器里，数据目录挂到环境卷。连接地址固定 `localhost:6379`/`localhost:3306`，零配置。不需要独立中间件容器和 Docker Network。

### 6. 部署边界

子智能体只做开发+编译+自测，容器用完即弃。生产部署由人工完成。

## 技术栈

| 层 | 技术 |
|----|------|
| 前端 | React + Vite + Ant Design + ReactFlow（DAG可视化） |
| 后端 | Python / FastAPI |
| 群主调度 | LangGraph（DAG状态图引擎） |
| 群主 LLM | LangChain + ChatAnthropic（结构化输出+多模型支持） |
| 子智能体运行时 | Claude Code CLI/SDK |
| 容器 | Docker（Ubuntu + Claude Code CLI 单一基础镜像） |
| 中间件 | Redis / MySQL（Claude Code 运行时自装自起，用户不感知） |
| 消息 | 消息总线（进程内，后续可换 RabbitMQ） |
| 存储 | 本地磁盘 Docker Volume（MVP），后续可换 PVC |
| DAG 可视化 | LangGraph 图结构 → API → ReactFlow 渲染 |
| 数据库 | PostgreSQL + Redis（docker-compose 自带，或连接已有实例） |
