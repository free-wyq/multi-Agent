# Multi-Agent 协作桌面应用

多智能体协作框架，解决软件工程虚拟交付问题。

## 项目简介

一个多智能体协作桌面应用，用户可以创建智能体、配置角色和技能，将智能体拉入群组协作完成软件交付任务。群主智能体负责意图分析和任务调度，子智能体通过本地 Claude Code CLI 执行开发、编译、测试等具体工作。

**核心定位**：桌面端工具，双击即用，零基础设施。

## 核心设计决策

### 1. 两类智能体

| | 群主 | 子智能体 |
|--|------|---------|
| 本质 | LLM API 直调 | Claude Code CLI 进程 |
| 需要沙箱 | ❌ | ❌ 本地进程 |
| 职责 | 意图分析、任务拆解、DAG调度 | 开发、编译、测试 |
| 运行位置 | Main 进程内 | child_process.spawn |
| 成本 | 低 | 中 |

### 2. 去掉 Docker

子智能体都是 Claude Code CLI 实例，不需要不同环境——只需要不同的 system prompt 和工作目录。本地进程比 Docker 容器启动更快，文件共享更简单（同一目录）。

### 3. 去掉 PostgreSQL + Redis

单机桌面应用，数据全在内存，持久化用 JSON 文件。进程内 EventEmitter 替代 Redis Pub/Sub。不需要查询优化、事务、跨进程通信。

### 4. 去掉 LangGraph + LangChain

LangGraph 的状态图在单进程内无优势。改为显式 async 方法 + while 循环，更易调试。LLM 调用改为直接 HTTP 请求，去除 LangChain 封装层。

### 5. DAG 依赖感知调度

无依赖的任务并行派发，有依赖的等前置完成后再派发。monitor 轮询 store 检查任务状态，推进下游。

### 6. @mention 智能路由

群聊消息中的 @mention 自动路由到对应智能体引擎。30 秒防循环机制，避免两个智能体互相 @ 死循环。

## 技术栈

| 层 | 技术 |
|----|------|
| 桌面框架 | Electron |
| 前端 | React + Vite + Ant Design + ReactFlow |
| 主进程 | Node.js / TypeScript |
| 群主调度 | async 方法 + while 循环（替代 LangGraph） |
| 群主 LLM | OpenAI 兼容 HTTP API 直调 |
| 子智能体运行时 | 本地 Claude Code CLI（child_process.spawn） |
| 数据存储 | 内存 Map + JSON 文件 |
| 事件总线 | Node.js EventEmitter |
| 进程间通信 | Electron IPC |
| DAG 可视化 | Coordinator 生成 → IPC → ReactFlow 渲染 |
| 跨平台 | macOS / Windows / Linux（electron-builder） |

## 快速开始

```bash
# 安装依赖
npm install

# 开发模式（启动 Electron + Vite）
npm run dev

# 打包桌面应用
npm run electron:build
```

## 环境要求

- Node.js 20+
- Claude Code CLI 已安装（或设置 `CLAUDE_CODE_PATH` 环境变量）
- LLM API 密钥（OpenAI / DeepSeek / 其他兼容端点）

## 项目结构

```
multi-Agent/
  electron/
    main.ts                     # Electron 主进程入口
    preload.ts                  # preload 脚本，暴露 IPC API
  src/                          # Renderer 进程（React 前端）
    pages/                      # 页面组件
    components/                 # 通用组件
    services/api.ts             # IPC API 调用层
    hooks/useWebSocket.ts       # 实时事件 hook
    ipc/                        # IPC 通道定义 + 类型
  main/                         # 主进程业务逻辑
    store/                      # 内存状态 + JSON 持久化
    bus/                        # EventEmitter 事件总线
    coordinator/                # 工作流 + LLM + 提示词
    agent-engine/               # 智能体引擎 + 大脑 + 注册表
    runtime/                    # Claude Code CLI 进程管理
    ipc-handlers/               # IPC 处理器
  data/                         # 运行时数据（JSON + 群组文件）
  public/                       # 静态资源
```
